# Сетка галереи / каталога — план R1.5 (после данных инвентаря)

> **Статус:** обновлён 2026-06-08 после повторного запуска панели с реальными данными инвентаря (17 картин, 2 класса размеров, все доступные по $400, 6 проданных / 11 в наличии, смесь портрет/ландшафт, /postcards сейчас пуст). Синтез R1 опирался на предположения, которые не выдержали столкновения с реальными данными; этот документ заменяет его. Где R1 пересмотрен — отмечено по тексту.

**Объект работы:** система `.gallery-grid` + `cardHTML()` в `index.html`, во всех 5 контекстах (главная — избранные, /works, /postcards, /collection/{name}, /past).

**Планка качества:** «Посетитель, скользящий взглядом по сетке, ощущает индивидуальность каждой картины, листает с удовольствием и естественно останавливается на одной, чтобы открыть её или написать — и никогда не чувствует, что покупает в ленте магазина».

**Вне зоны охвата:** внутренности модала, hero, навигация, контент/данные картин, бэкенд photo-manager.

---

## Реальный инвентарь

- **17 картин всего** (не «50+»).
- **11 в наличии + 6 проданных.**
- **2 класса размера:** 16×20 in (12 картин, ~70%) и 11×14 in (5, ~30%).
- **Все доступные картины с одинаковой ценой $400.** У проданных цена не указана.
- **Смесь портрет/ландшафт** → ~4 разных форм карточки (16×20p, 20×16l, 11×14p, 14×11l).
- **Все картины — акрил на холсте.** В инвентаре нет акварелей → **/postcards рендерится пустой.**
- **5 коллекций:** Seasons, Flowers, Big City, Forest (и другие), 1–2 тега на картину.
- **Год редко заполнен:** только у 4 из 17 картин (2024 или 2025).
- **Колонка `dominantColor`** упоминается в коде, но в таблице не заполнена — цвето-ориентированная кураторская работа упирается в данные.

## Разрешённые конфликты (после данных; **жирным** — изменения относительно R1)

### 1. Каталожный № — **УДАЛИТЬ везде** ⚠️ *отменяет решение R1*

- **Позиция R1:** сохранить, ослабить, перенести в угловой overlay, скрыть на главной + /past.
- **Позиция R1.5:** Покупатель + Скептик + Аудитор усилили аргумент за удаление (малое N делает № бесполезным для навигации и редакторски избыточным — «как нумеровать стулья в собственной гостиной»); Куратор + Дизайнер продолжали отстаивать сохранение.
- **Решение пользователя:** удалить везде.
- **Что делаем:** убрать CSS-блок `.card-number` (`index.html:441-451`); убрать слот `${num}` из `cardHTML` (`:1932-1937`); убрать рендер `heroNum` в `renderHero` (~`:2270-2271`); заменить курсорные подписи `View №XX` (~`:2928-2934`) на `View "${title}"`. Поле `catalogNumber` оставить только в данных — при необходимости вывести в модал.

### 2. Цена на карточке — **сохранить, перенести под подпись** ⚠️ *пользователь отменил консенсус R1.5*

- **Консенсус R1.5** (Покупатель + Скептик + Дизайнер + Аудитор): убрать цену с карточки, упомянуть один раз в шапке /works (`Все картины $400 · 16×20 или 11×14 in · акрил на холсте`). 11 одинаковых $400 карточек читается как товар.
- **Решение пользователя:** оставить и подпись, и цену на карточке; справиться типографикой (позиция R1).
- **Что делаем:** остаёмся в R1 — `.card-info { display: block }`, `.card-price { display: block; margin-top: 0.4rem; font-style: italic; color: var(--text-secondary); }`. `Sold` рендерим в том же слоте через `.card-price.card-price--sold`. Убрать встроенный `soldTag` из `.card-title`. **Применяется уточнение Дизайнера из R1.5:** уменьшить вес цены (`font-size: 0.88rem`, цвет `var(--text-faint)`) — равномерная цена = неинформативна; слот существует ради консистентности раскладки, а не сигнала.

### 3. Masonry — **сохранить + interleave Дизайнера + 85% врезка для 11×14** (предварительно; пересматриваем) ⚠️ *новое из R1.5*

- **Позиция R1:** сохранить masonry, починить FOUC.
- **Позиция R1.5:** Скептик усилил аргумент за равномерные плитки («Tetris-автопилот — 4 формы никогда полностью не сходятся»); Дизайнер предложил спасти masonry чередованием портрет/ландшафт в порядке и врезкой 11×14 до 85% ширины колонки (тонкий сигнал реального масштаба без чипа); Куратор + Разработчик держали линию за сохранение реальных пропорций.
- **Решение пользователя:** вариант Дизайнера на сейчас, не финальный.
- **Что делаем:**
  - Сохраняем masonry. FOUC чиним фиксом из R1 (засев `grid-row: span N` на этапе рендера из `aspectRatioFor(p)`; стартуем с `grid-auto-rows: 8px` сразу; без `.masonry-ready` swap). При 4 известных aspect ratios значения span — маленький lookup-table.
  - На уровне данных (где `PAINTINGS` сортируется перед рендером): чередуем портреты/ландшафты так, чтобы подряд шли не более 2 карточек одной ориентации. Где у Светы есть кураторский порядок — он побеждает; но в качестве дефолтной функции упорядочивания — interleave.
  - Карточки 11×14 рендерим в ~85% ширины колонки, по центру: `.painting-card[data-size="small"] .card-image { width: 85%; margin: 0 auto; }`. Заголовок / подпись / цена остаются на полную ширину. Тонкая врезка = подсказка о реальном масштабе без чипа `S/M`.
  - **Можно откатить:** если результат всё ещё читается как «uncanny valley» (риск Скептика), запасной вариант — строгая сетка с разбиением по ориентации (см. ниже).

### 4. UI сортировки / фильтра на /works — **никакого UI; разбиение по размеру** (вместо разбиения по году)

- **R1.5:** Покупатель сам снял свой R1-запрос на полосу сортировки («11 карточек, все $400, sort UI бесполезен на этом масштабе»); идея Куратора с разбиением по году умерла (данные о годе слишком редки); заменена разбиением по размеру (Большие 16×20 / Маленькие 11×14). Аудитор добавляет: это также даёт /works структуру `<h2>`, которую требует WCAG 2.4.10.
- **Что делаем:** в `renderPaintings` (~`:2498`) разделить `works` по классу размера; рендерить два `<section aria-labelledby="grp-N"><h2 id="grp-N">Большие работы — 16 × 20 in</h2><div class="gallery-grid">…</div></section>` блока. Визуально сдержанно (display-italic заголовок, hairline сверху, ~3rem margin-top). Один и тот же `cardHTML` в обеих сетках.

### 5. Аффорданс карточки — **`<button>` → `<a href="#painting/${id}">`** (без изменений)

- Единогласно; опорное требование Покупателя.
- **Что делаем:** рендерим карточку как `<a href="#painting/${p.id}" onclick="event.preventDefault(); openModal(${p.id});">`. Пушим history state в `openModal`. Вешаем `popstate` listener.
- **Семантика popstate (предлагаемый дефолт):**
  - Клик по карточке → `history.pushState({ paintingId: id }, '', '#painting/' + id)`; открыть модал.
  - Закрытие модала → `history.back()`, если `state.paintingId` совпадает; иначе `history.replaceState({}, '', window.location.pathname + window.location.search)`.
  - `popstate` → если хеш совпадает с id картины, открыть тот модал; если хеша картины нет — закрыть открытый модал.
  - Заход по прямой ссылке (`/#painting/5`) → модал открывается на первом рендере; закрытие заменяет хеш без ухода с сайта.

### 6. Featured grid на главной — **сократить с 8 до 6** ⚠️ *новое из R1.5*

- **R1.5:** Скептик отметил, что 8 из 17 (47%) читается как preview, а не отбор. Мягкий ход Куратора: сократить до 6, чтобы у /works было больше «открытия»; жёсткий ход Скептика: полностью убрать двойную сетку.
- **Решение пользователя:** 6.
- **Что делаем:** `selectFeatured(PAINTINGS, 6, …)` в `:2312`. Также убираем ветку padding в `selectFeatured` (~`:2196-2199`) — доступных всего 11; padding проданными работами срабатывает слишком часто. Ограничиваем `Math.min(6, availablePool.length)`.

### 7. Страница /postcards — **скрыть пункт меню, пока пусто** ⚠️ *новое из R1.5*

- **R1.5:** Куратор + Покупатель + Скептик + Аудитор отметили это как нарушение Nielsen #1/#5 — пункт меню ведёт на извиняющееся пустое состояние без пути восстановления.
- **Что делаем:** в рендере навигации (~`:1529, :2857`) ставим условие на пункт `<li>` Postcards: `PAINTINGS.some(p => p.medium === 'watercolor' || p.medium === 'watercolour' || (p.collections||[]).includes('postcards'))`. Маршрут остаётся доступен по URL — когда инвентарь появится. Одно условие; убирает тупик.
